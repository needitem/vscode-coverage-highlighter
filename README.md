# Coverage Highlighter

`Coverage Highlighter`는 Coverage XML 파일을 읽어 VS Code 에디터에 커버리지 상태를 표시하고, 미커버 코드를 분류하고, 보고서까지 생성하는 확장 프로그램입니다.

이 확장은 단순히 "빨간 줄 보기"에서 끝나지 않습니다. 미커버 라인을 `문서`, `주석 예정`, `커버 예정`으로 나눠 관리하고, 나중에 보고서로 정리할 수 있게 설계되어 있습니다.

## 무엇을 할 수 있나

- Coverage XML을 읽어 에디터에 라인 단위로 하이라이트 표시
- 미커버 라인을 트리 뷰에서 파일별로 확인
- 연속된 미커버 라인을 블록 단위로 분류
- `문서`, `주석 예정`, `커버 예정` 카테고리로 관리
- 문서 분류 시 사유(reason)까지 저장
- 다중 선택 후 일괄 분류, 수정, 삭제
- 검색, 정렬, 최근 XML 재사용
- Markdown 보고서 생성

## 지원 포맷

- VectorCAST Coverage XML
- 루트 노드가 `cv.CoverResult`인 XML

## 화면 구성

확장을 사용하면 주로 아래 3곳을 보게 됩니다.

### 1. 에디터 하이라이트

- 녹색: 커버됨
- 빨간색: 미커버
- 노란색: 부분 커버

### 2. 상태바

- 우측 상태바에 전체 구문 커버리지가 표시됩니다.
- 상태바를 클릭하면 `Coverage 요약` 웹뷰를 열 수 있습니다.

### 3. Coverage 탐색기

Activity Bar의 `Coverage Highlighter` 아이콘을 열면 `Coverage 탐색기`가 나타납니다.

탐색기는 다음 3개의 루트로 구성됩니다.

- `작업 및 최근 XML`
- `미분류`
- `분류됨`

## 설치

현재 저장소 기준으로는 로컬 설치나 VSIX 설치 기준으로 사용하는 것이 가장 확실합니다.

### VSIX로 설치

1. 저장소 루트에서 의존성을 설치합니다.
2. VSIX 파일을 생성합니다.
3. VS Code에서 `Extensions: Install from VSIX...`를 실행합니다.

```bash
npm install
npx @vscode/vsce package
```

생성된 파일 예:

```text
coverage-highlighter-1.15.5.vsix
```

### 개발 모드로 실행

```bash
npm install
npm run compile
```

이후 VS Code에서 확장 개발 호스트로 실행하면 됩니다.

## 빠른 시작

가장 기본적인 사용 순서는 아래와 같습니다.

1. 워크스페이스를 엽니다.
2. `Coverage: XML 불러오기`를 실행합니다.
3. 에디터에서 커버리지 하이라이트를 확인합니다.
4. `Coverage 탐색기`의 `미분류` 항목에서 미커버 라인을 확인합니다.
5. 필요한 라인을 `문서`, `주석 예정`, `커버 예정`으로 분류합니다.
6. 필요하면 `Coverage: 분류 보고서 생성`으로 Markdown 보고서를 만듭니다.

## 상세 사용법

### 1. Coverage XML 불러오기

다음 방법 중 하나로 XML을 불러올 수 있습니다.

- Command Palette에서 `Coverage: XML 불러오기`
- `Coverage 탐색기`의 `작업 및 최근 XML` 아래 `XML 로드`
- 최근 불러온 XML 목록에서 재선택

불러오기가 완료되면 다음 동작이 자동으로 수행됩니다.

- XML 파싱
- 열린 에디터들에 하이라이트 적용
- 상태바 업데이트
- `Coverage 탐색기` 갱신
- 최근 XML 경로 캐시 저장

### 2. 파일 매칭 방식

XML에 들어 있는 경로와 로컬 워크스페이스 경로가 완전히 같지 않아도, 확장은 파일명과 경로 suffix를 기준으로 매칭을 시도합니다.

즉 다음과 같은 경우에도 매칭될 수 있습니다.

- XML은 절대경로
- 로컬은 다른 드라이브나 다른 루트 경로
- 파일명은 같고 하위 경로 구조가 유사한 경우

그래도 매칭이 안 되면 먼저 아래를 확인하는 것이 좋습니다.

- 해당 파일이 현재 워크스페이스 안에 있는지
- 파일명이 실제로 같은지
- XML이 최신 빌드 산출물인지

### 3. 하이라이트 읽는 법

- 녹색 줄은 이미 커버된 라인입니다.
- 빨간 줄은 전혀 커버되지 않은 라인입니다.
- 노란 줄은 부분 커버된 라인입니다.

분류되지 않은 미커버 라인만 집중해서 보고 싶다면 `Coverage 탐색기`에서 `분류된 항목 숨기기`를 사용할 수 있습니다. 이 옵션을 켜면 분류된 미커버 라인은 에디터 강조 대상에서도 제외됩니다.

### 4. Coverage 탐색기 사용법

#### `작업 및 최근 XML`

여기에는 자주 쓰는 액션과 최근 XML 목록이 들어 있습니다.

- `XML 로드`
- 최근 불러온 XML 다시 열기
- `분류된 항목 숨기기` 또는 `분류된 항목 보기`
- `사유 관리`
- `보고서 생성`

#### `미분류`

- 아직 분류되지 않은 미커버 라인만 보여줍니다.
- 파일별로 묶여 표시됩니다.
- 연속된 라인은 블록으로 합쳐서 보여줍니다.
  - 예: `42-47라인`
- 블록을 열면 바로 분류 가능한 옵션이 나옵니다.

#### `분류됨`

- `문서`, `주석 예정`, `커버 예정` 카테고리별로 묶입니다.
- 문서 분류는 다시 `사유`별로 하위 그룹이 생깁니다.
- 파일과 라인 단위로 내려가며 확인할 수 있습니다.

### 5. 현재 라인 또는 선택 영역 분류

에디터 안에서 바로 분류할 수도 있습니다.

#### 현재 라인 분류

- Command Palette: `Coverage: 현재 미커버 라인 분류`
- 또는 에디터 우클릭 메뉴

커서가 위치한 라인이 연속 미커버 블록에 속해 있으면, 한 줄만이 아니라 해당 블록 전체가 같이 분류됩니다.

#### 선택 라인 분류

- Command Palette: `Coverage: 선택 라인 분류`
- 또는 에디터에서 범위를 선택한 뒤 우클릭 메뉴

선택 범위 안에 포함된 미커버 블록들을 모아 한 번에 분류합니다.

### 6. 트리 뷰에서 분류하기

`미분류` 아래 블록을 열면 바로 분류 가능한 액션이 표시됩니다.

- `문서: <사유>`
- `문서: 새 사유 추가...`
- `주석 예정`
- `커버 예정`

문서 카테고리는 사유를 함께 저장합니다. 반면 `주석 예정`, `커버 예정`은 사유 없이 바로 분류됩니다.

### 7. 카테고리 의미

#### 문서

- 보고서에 포함할 라인
- 왜 커버되지 않는지 설명이 필요한 코드
- 사유를 함께 저장

예:

- 외부 장비 의존 코드
- 예외 처리 방어 코드
- UI 진입 불가 경로

#### 주석 예정

- 당장 제거하지는 않지만 추후 주석 처리하거나 비활성화할 코드

#### 커버 예정

- 추후 테스트 추가나 시나리오 보강으로 커버리지를 올릴 대상

### 8. 사유 관리

문서 카테고리에서 사용하는 사유는 별도로 관리할 수 있습니다.

실행 방법:

- `Coverage: 사유 관리`
- 또는 `Coverage 탐색기`의 `사유 관리`

가능한 작업:

- 새 사유 추가
- 기존 사유 삭제

주의:

- 사유를 삭제해도 이미 저장된 분류 항목의 라벨이 자동으로 바뀌지는 않습니다.
- 사유 목록에서만 제거됩니다.

### 9. 빠른 분류 단축키

자주 쓰는 분류는 단축키로 바로 처리할 수 있습니다.

| 단축키 | 기본 동작 |
| --- | --- |
| `Ctrl+Shift+1` | 문서 분류 |
| `Ctrl+Shift+2` | 주석 예정으로 분류 |
| `Ctrl+Shift+3` | 커버 예정으로 분류 |
| `Ctrl+Shift+4` ~ `Ctrl+Shift+9` | 사용자 정의 빠른 분류 슬롯 |

빠른 슬롯은 `coverageHighlighter.quickSlot1` ~ `coverageHighlighter.quickSlot9` 설정으로 제어합니다.

예시:

```json
{
  "coverageHighlighter.quickSlot1": {
    "category": "document",
    "reason": "외부 장비 의존"
  },
  "coverageHighlighter.quickSlot2": {
    "category": "comment-planned",
    "reason": ""
  },
  "coverageHighlighter.quickSlot3": {
    "category": "cover-planned",
    "reason": ""
  },
  "coverageHighlighter.quickSlot4": {
    "category": "document",
    "reason": "방어 로직"
  }
}
```

동작 참고:

- `document`인데 `reason`이 비어 있으면 실행 중 사유를 다시 묻습니다.
- `comment-planned`, `cover-planned`는 사유 없이 바로 분류됩니다.

### 10. 검색과 정렬

`Coverage 탐색기` 상단 액션으로 미분류 목록을 빠르게 좁힐 수 있습니다.

#### 파일 검색

- 명령: `Coverage: 파일 검색`
- 파일명, 전체 경로 일부, 확장자 등을 기준으로 필터링

예:

- `Controller`
- `src/main`
- `.cpp`

#### 정렬

- 명령: `Coverage: 정렬`

지원 정렬:

- 파일명 오름차순/내림차순
- 미분류 개수 적은 순/많은 순
- 경로 오름차순/내림차순

#### 검색 초기화

- 명령: `Coverage: 검색 초기화`

### 11. 기존 분류 수정/삭제

`분류됨` 아래의 라인 항목은 수정과 삭제가 가능합니다.

가능한 작업:

- `분류 수정`
- `분류 삭제`
- `선택 항목 일괄 수정`
- `선택 항목 일괄 삭제`

다중 선택 후 컨텍스트 메뉴를 사용하면 여러 라인을 한 번에 정리할 수 있습니다.

### 12. 전체 분류 초기화

저장된 분류를 전부 지우고 싶다면 아래 명령을 사용합니다.

- `Coverage: 분류 전체 초기화`

이 작업은 되돌릴 수 없으므로 주의가 필요합니다.

### 13. 요약 보기

전체 파일 수준에서 커버리지를 훑고 싶다면 다음 방법을 사용합니다.

- 상태바 클릭
- Command Palette에서 `Coverage: 요약 보기`

요약 화면에서는 아래 정보를 볼 수 있습니다.

- 전체 구문 커버리지
- 전체 분기 커버리지
- 파일 수
- 파일별 커버 라인 / 부분 커버 라인 / 미커버 라인 / 커버율

파일 목록은 커버율이 낮은 항목부터 우선 정렬되어, 우선 대응 대상을 빠르게 확인할 수 있습니다.

### 14. 분류 보기

분류된 항목을 웹뷰 형태로 한 번에 보고 싶다면 다음 명령을 사용합니다.

- `Coverage: 분류 보기`

이 화면은 카테고리와 사유 기준으로 묶인 Markdown 스타일 테이블에 가까운 요약 뷰입니다.

### 15. 보고서 생성

다음 명령으로 Markdown 보고서를 만들 수 있습니다.

- `Coverage: 분류 보고서 생성`

보고서 종류:

- `문서 보고서`
- `전체 보고서`

결과는 새 Markdown 문서로 열립니다.

보통은 아래 식으로 사용하면 됩니다.

- 외부 공유용이면 `문서 보고서`
- 내부 추적용이면 `전체 보고서`

## 추천 사용 흐름

실무에서는 아래 흐름이 가장 효율적입니다.

1. 테스트 실행 후 Coverage XML 생성
2. `Coverage: XML 불러오기`
3. `미분류`에서 빨간 블록 우선 검토
4. 당장 테스트할 수 없는 코드는 `문서`
5. 죽은 코드나 후속 정리 대상은 `주석 예정`
6. 곧 테스트 추가할 대상은 `커버 예정`
7. 필요 시 `파일 검색`과 `정렬`로 범위를 좁힘
8. 마지막에 `분류 보고서 생성`

## 설정

### 색상 설정

```json
{
  "coverageHighlighter.coveredColor": "rgba(0, 255, 0, 0.2)",
  "coverageHighlighter.uncoveredColor": "rgba(255, 0, 0, 0.2)",
  "coverageHighlighter.partialColor": "rgba(255, 255, 0, 0.2)"
}
```

### 빠른 분류 슬롯 설정

```json
{
  "coverageHighlighter.quickSlot1": {
    "category": "document",
    "reason": ""
  },
  "coverageHighlighter.quickSlot2": {
    "category": "comment-planned",
    "reason": ""
  },
  "coverageHighlighter.quickSlot3": {
    "category": "cover-planned",
    "reason": ""
  }
}
```

가능한 category 값:

- `document`
- `comment-planned`
- `cover-planned`

## 저장과 캐시

확장은 사용 편의를 위해 일부 상태를 자동 저장합니다.

- 최근에 연 XML 목록
- 현재 XML 경로
- 문서 편집으로 인해 달라진 라인 오프셋

캐시 파일은 워크스페이스 기준으로 아래 경로에 저장됩니다.

```text
.vscode/coverage-cache.json
```

즉 워크스페이스를 닫았다가 다시 열어도 최근 XML이나 일부 라인 보정 정보가 유지됩니다.

## 문제 해결

### XML은 불러왔는데 아무 파일도 하이라이트되지 않음

아래를 확인하세요.

- 워크스페이스가 실제 소스 루트를 포함하는지
- XML 안의 파일 경로와 로컬 파일명이 어느 정도 대응되는지
- XML이 오래된 산출물은 아닌지

### 라인 번호가 맞지 않는 것 같음

확장은 편집 중 라인 오프셋을 추적하지만, 원본 코드가 크게 바뀐 경우에는 XML을 다시 생성해서 불러오는 편이 정확합니다.

### 문서 분류 시 계속 사유를 물어봄

빠른 슬롯에 `document`를 사용하면서 `reason`을 비워둔 경우입니다. 설정에 기본 사유를 넣으면 바로 분류할 수 있습니다.

### 최근 XML이 오래된 파일을 가리킴

`Coverage: XML 불러오기`에서 새 파일을 다시 선택하면 최근 목록이 갱신됩니다.

## 명령 요약

자주 쓰는 명령은 아래와 같습니다.

| 명령 | 용도 |
| --- | --- |
| `Coverage: XML 불러오기` | Coverage XML 로드 |
| `Coverage: 하이라이트 초기화` | 현재 하이라이트 제거 |
